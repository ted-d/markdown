Это _тоже курсив_ с помощью подчеркиваний. ``` _тоже курсив_ ```


Это **жирный** текст с помощью звездочек. ``` **жирный**```

Это __тоже жирный__ текст с помощью подчеркиваний. ``` __тоже жирный__ ```

А это ***жирный курсив*** с помощью звездочек. ``` ***жирный курсив***```

И это ___тоже жирный курсив___ с помощью подчеркиваний.  ``` ___тоже жирный курсив___ ```

Формально можно и **_комбинировать_** оба символа (хотя, это ухудшает читабельность текста). ``` **_комбинировать_**```

⚠️ Важно: символы выделения должны *примыкать к тексту* без пробелов. ``` *примыкать к тексту*```

# Разделительные линии

Конспект урока
Горизонтальная линия (или разделитель) — это элемент, который визуально разделяет разные разделы документа длинной горизонтальной чертой. Она помогает структурировать контент, обозначая крупные смысловые блоки, и улучшает визуальное восприятие текста.

 

Как это работает?
Для создания линии в Markdown используются три одинаковых символа подряд на отдельной строке. Пробелы между символами не обязательны, но улучшают читаемость исходного кода. Допустимые символы: дефисы (-), звёздочки (*) или знаки подчёркивания (_).

 

Почему это важно?
Разделительная линия — это мощный визуальный инструмент для организации контента. Она особенно полезна в длинных документах, где помогает читателю понять, где заканчивается одна тема и начинается другая, не перегружая текст лишними заголовками.

 

Исходный код (Markdown):
Это первый раздел документа. ```***```

***

Вот так создается простая разделительная линия.```---```

---

Можно использовать дефисы (самый популярный вариант).```___```

___

Или нижние подчеркивания. ```- - -```

- - -

Между символами можно ставить пробелы.```* * *```

* * *

Это тоже будет работать.

Второй раздел документа после линии.
 

Важное примечание: Для гарантированного и предсказуемого создания линии настоятельно рекомендуется делать пустые строки до и после строки с символами. Хотя многие парсеры Markdown (особенно современные) справятся и без них, отсутствие пустых строк может привести к непреднамеренному результату.

# Программный код

Markdown предоставляет мощные инструменты для вставки программного кода, обеспечивая его корректное и читаемое отображение. Поддержка кода разделяется на три основные области:

Строчные (inline) фрагменты — для коротких упоминаний в теле текста.

Многострочные блоки — для демонстрации больших примеров.

Подсветка синтаксиса — автоматическое раскрашивание кода в зависимости от языка программирования для улучшения восприятия.

 

## Как это работает?

Строчные фрагменты обрамляются одинарными бэктиками (`).

Блоки кода обрамляются тройными бэктиками (```) или создаются через отступ (4 пробела или 1 табуляция).

Подсветка синтаксиса активируется указанием имени языка программирования сразу после открывающих тройных бэктиков (напр., ````python`).

 

## Почему это важно?
Корректное оформление кода — признак качественной технической документации. Оно предотвращает непреднамеренную интерпретацию символов Markdown, сохраняет исходное форматирование и многократно повышает читаемость и удобство восприятия сложных примеров.

# Экранирование
Обратный слеш или бэкслеш (\) в Markdown является экранирующим символом. Его главная задача — сообщить парсеру, что следующий за ним символ нужно рассматривать как обычный текст, а не как часть разметки.

 

Как это работает?
Вы ставите символ \ непосредственно перед тем символом Markdown, который должен отобразиться в чистом виде. Это отменяет его специальное значение.

 

Почему это важно?
Многие символы в Markdown имеют специальное значение: *, _, `, ~, #, +, -, ., !, [, ], (, ), {, }, |. Если вам нужно написать их как часть текста, экранирование — единственный надежный способ сделать это правильно и избежать непредвиденного форматирования.

 

Примеры синтаксиса и результата
Исходный код (Markdown):

# Без экранирования:
Это *курсив* ```*курсив*```

, а это **жирный**. ```*жирный**```

# С экранированием:
Это \*не курсив\*, а это \*\*не жирный\*\*.

Символ решётки \# не сделает заголовок.
Скобки \[вот так\] и \(вот так\).
Математическое выражение: 2 \* 2 = 4.

Экранирование самого обратного слеша: \\ -> \
 

Как это будет выглядеть после обработки:

Без экранирования:
Это курсив, а это жирный.

С экранированием:
Это *не курсив*, а это **не жирный**.

Символ решётки # не сделает заголовок.
Скобки [вот так] и (вот так).
Математическое выражение: 2 * 2 = 4.

Экранирование самого обратного слеша: \ -> \

 

Важные нюансы:

Внутри блоков кода (как строчных, так и блочных) экранирование не нужно, так как всё их содержимое уже выводится "как есть".

Не все символы требуют обязательного экранирования. Например, точка (.) или восклицательный знак (!) чаще всего не несут специального значения в середине текста, но их экранирование не навредит.

# Цитаты
Цитаты (или блоки цитирования) — это элемент разметки для выделения цитируемого текста из другого источника. Визуально они обычно отображаются с отступом, иногда с вертикальной полосой слева и/или изменённым стилем шрифта (например, курсивом).

 

Как это работает?
Для создания цитаты в Markdown используется символ «больше» (>), помещённый в начале строки перед текстом. Цитаты можно делать многоуровневыми (вкладывать друг в друга) и комбинировать с другой разметкой внутри (списки, код, заголовки).

 

Почему это важно?
Цитаты семантически и визуально отделяют чужую мысль или важную выдержку от основного текста. Это повышает удобочитаемость, показывает уважение к авторству и помогает структурировать сложные материалы, где важно разграничить свою и чужую точку зрения.

 

Примеры синтаксиса и результата
Исходный код (Markdown):
``` text
> Это простой пример цитаты. Весь этот текст будет выделен в особый блок.
>
> Цитата может состоять из нескольких абзацев. Для этого между ними нужно оставить пустую строку тоже с символом `>`.
 ```

Как это будет выглядеть после обработки:

> Это простой пример цитаты. Весь этот текст будет выделен в особый блок.
>
> Цитата может состоять из нескольких абзацев. Для этого между ними нужно оставить пустую строку тоже с символом >.

# Ссылки

Ссылки (гиперссылки) — это способ связать текст вашего документа с другим ресурсом в сети (веб-страницей, изображением, файлом) или с другим разделом того же документа.

 

Как это работает?
В Markdown существует несколько способов создания ссылок, но все они состоят из двух частей:

Анкор (anchor) — видимый, кликабельный текст.

URL (Uniform Resource Locator) — адрес, на который ведёт ссылка.

 

Почему это важно?
Ссылки — это основа интернета. Они позволяют создавать перекрёстные ссылки между документами, ссылаться на источники, предоставлять дополнительные материалы и создавать навигацию внутри длинных документов.

 

Примеры синтаксиса и результата
1. Простая inline-ссылка (самый частый способ)

Синтаксис: ```[анкор](URL "Необязательный title")```

Исходный код (Markdown):
```
Это [ссылка на Google](https://www.google.com "Крупнейшая поисковая система") с title -  всплывающая приписка снизу.
А это [ссылка без title](https://example.com).
```

Как это будет выглядеть после обработки:

Это [ссылка на Google](https://www.google.com "Крупнейшая поисковая система") с title.


А это [ссылка без title](https://example.com).

 

2. Автоматические ссылки

Синтаксис: Обрамление URL-адреса или email-адреса в угловые скобки < >.

Исходный код (Markdown):
```
Наш сайт: <https://www.example.com>
Свяжитесь с нами: <contact@example.com>
```
Как это будет выглядеть после обработки:

Наш сайт: <https://www.example.com>

Свяжитесь с нами: <contact@example.com>

 

3. Якоря (ссылки внутри документа)

Синтаксис: Необходимо сначала создать якорь с помощью атрибута id (часто через HTML, т.к. в чистом Markdown нет единого стандарта), а затем ссылаться на него.

Исходный код (Markdown):
```
<!-- Создание якоря (часто делается через HTML) -->
<span id="chapter1"></span>

## Глава 1

...

<!-- Ссылка на якорь -->
[Вернуться к Главе 1](#chapter1)
```

# Изображения 
Вставка изображений в Markdown работает по принципу, очень похожему на создание ссылок, но с восклицательным знаком (!) в начале. Это позволяет добавлять визуальный контент в документ.

 

Как это работает?
Синтаксис для изображения аналогичен синтаксису ссылки, но перед квадратными скобками ставится восклицательный знак !.

```
![Альтернативный текст](URL-адрес_изображения "Необязательный заголовок")
```

![Альтернативный текст](URL-адрес_изображения "Необязательный заголовок")
 

Почему это важно?
Изображения делают документацию, статьи и README-файлы наглядными. Они помогают проиллюстрировать сложные концепции, показать интерфейс программы, результаты работы кода или просто сделать документ визуально привлекательнее.

 

Примеры синтаксиса и результата
1. Простая inline-вставка изображения (самый частый способ)

Синтаксис: ```![Альтернативный текст](URL "Необязательный title")```

Исходный код (Markdown):

![Логотип GitHub](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png "GitHub")
 

Как это будет выглядеть после обработки:

(Будет вставлено изображение логотипа GitHub с подписью "Логотип GitHub". При наведении курсора появится подсказка "GitHub").

 

2. Ссылка-изображение (Изображение, являющееся гиперссылкой)

Синтаксис: Комбинация синтаксиса ссылки и изображения. Обёртывание синтаксиса изображения в синтаксис ссылки. ```[![]()]()```

```[![Альтернативный текст](URL_изображения)](URL_ссылки)```

Исходный код (Markdown):

(Будет показано изображение-кнопка. При клике на него произойдет переход на google.com).

# Чекбокс

Чекбоксы (или флажки) в Markdown — это особый вид элемента списка, который позволяет создавать интерактивные (или псевдо-интерактивные) списки задач (checklists). Пользователь может отмечать выполненные пункты.

 

Как это работает?
Синтаксис основан на неупорядоченном списке, где вместо обычного маркера (-, *, +) используются квадратные скобки [ ] для неотмеченного пункта и [x] (или [X]) для отмеченного. Важно: Для работы интерактивности (возможности ставить галочки) часто требуется поддержка со стороны платформы (например, GitHub, GitLab, некоторые редакторы).

 

Почему это важно?
Чекбоксы невероятно полезны для создания:

Списков дел (to-do lists) в проектах.

Чек-листов для проверки выполнения требований.

Инструкций с пошаговыми действиями, которые можно отмечать.

Визуального отображения прогресса выполнения задачи.

 

Примеры синтаксиса и результата
Базовый список задач:

Исходный код (Markdown):
```
- [ ] Провести митинг в 10:00
- [ ] Написать отчет
- [x] Закончить работу над новой функцией
- [ ] Ответить на письма
```

# Таблицы

Таблицы в Markdown — это способ структурирования данных в строки и столбцы. Они позволяют организовать информацию в удобном для сравнения и анализа виде.

 

Как это работает?
Синтаксис таблиц использует вертикальные черты (|) для разделения столбцов и дефисы (-) для отделения строки заголовка от данных. Выравнивание содержимого столбцов задается с помощью двоеточия (:) в строке с дефисами.

 

Почему это важно?
Таблицы незаменимы для представления структурированных данных: сравнения характеристик, отображения расписаний, организации параметров, результатов тестов и многого другого. Они делают сложную информацию наглядной и легко воспринимаемой.

 

Примеры синтаксиса и результата
Базовая таблица:

Исходный код (Markdown):
```
| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|-------------|-------------|-------------|
| Ячейка 1    | Ячейка 2    | Ячейка 3    |
| Ячейка 4    | Ячейка 5    | Ячейка 6    |
```

Как это будет выглядеть после обработки:

| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|-------------|-------------|-------------|
| Ячейка 1    | Ячейка 2    | Ячейка 3    |

# Выравнивание текста в таблицах

Markdown позволяет задавать выравнивание содержимого в ячейках таблицы: по левому краю, по центру или по правому краю. Это удобно для числовых данных, дат или любой информации, которую важно оформить аккуратно.

 

Как это работает?
Выравнивание задаётся символами : в строке-разделителе под заголовками таблицы:

```:---``` — выравнивание по левому краю

```:---:``` — выравнивание по центру

```---:``` — выравнивание по правому краю

⚠️ Важно: двоеточия работают только в строке-разделителе, в остальных строках их использовать не нужно.

 

Почему это важно?

Делает таблицы более читаемыми.

Помогает правильно выстраивать текст и числа.

Позволяет создавать профессионально выглядящие документы.

 

Примеры синтаксиса и результата
Таблица с выравниванием:

Исходный код (Markdown):
```
| Левый край | По центру | Правый край |
| :--------- | :-------: | ----------: |
| Текст      | Текст     | Текст       |
| Ещё текст  | Ещё текст | Ещё текст   |
```



Как это будет выглядеть после обработки:
| Левый край | По центру | Правый край |
| :--------- | :-------: | ----------: |
| Текст      | Текст     | Текст       |
| Ещё текст  | Ещё текст | Ещё текст   |
 

# Комментарии

Комментарии в Markdown — это фрагменты текста, которые не должны отображаться в итоговом отрендеренном документе. Они нужны для того, чтобы оставлять заметки, пояснения или инструкции для тех, кто будет читать или редактировать исходный код Markdown-файла.

 

Как это работает?
В стандартном CommonMark/Markdown нет специального синтаксиса для комментариев. Однако на практике используются два основных подхода, которые поддерживаются большинством парсеров:

HTML-комментарии — наиболее универсальный и распространенный способ.

Специальная форма ссылки — альтернативный метод, который выглядит как ссылка-нотация.

 

Почему это важно?
Комментарии помогают:

Объяснять сложные участки разметки.

Оставлять TODO-заметки для будущих правок.

Временно "закомментировать" (скрыть) часть контента без его удаления.

Общаться с другими collaborators прямо в исходном коде файла.

 

Примеры синтаксиса и результата
1. HTML-комментарии (наиболее распространенный способ):

Исходный код (Markdown):
```
<!-- Это однострочный комментарий. Его не будет видно в готовом документе. -->
```
```
<!--
  Это многострочный комментарий.
  Он может занимать несколько строк.
  Идеально для больших пояснений.
-->
```


 

Специальная форма ссылки (альтернативный способ):

Исходный код (Markdown):

```
[comment]: # "Это комментарий в форме ссылки"
[//]: # (Еще один вариант синтаксиса)
[^_^]: # "Невидимый комментарий с смайликом"
```
Этот текст будет виден, а комментарии выше - нет.

```
[comment]: <> (Еще один вариант синтаксиса для комментария)
 ```



 

Особенности альтернативного способа:

Должен быть размещен на отдельной строке

Может использовать разные разделители: #, <>

Может использовать разные скобки: (), "", ''

Не поддерживается некоторыми парсерами (менее универсален, чем HTML-комментарии)

 

Важное примечание:

Комментарии остаются видимыми в исходном коде файла. Они не являются способом скрытия конфиденциальной информации.

HTML-комментарии — самый надежный и универсальный метод.

Альтернативный способ может быть полезен в специфических случаях или для создания "невидимых" сносок.

# Эмодзи

Эмодзи — это небольшие графические символы (смайлики, иконки, пиктограммы), которые используются для передачи эмоций, обозначения объектов или украшения текста. В Markdown есть несколько способов их вставки.

 

Как это работает?
Существует три основных способа использования эмодзи в Markdown:

Непосредственная вставка — копирование и вставка эмодзи прямо в текст (работает там, где поддерживается Unicode).

Кодовые последовательности — использование специального синтаксиса :имя_эмодзи:.

 

Почему это важно?
Эмодзи делают текст более живым и выразительным. Они особенно полезны в неформальной документации, README-файлах, комментариях и сообщениях, где помогают передать эмоциональную окраску, привлечь внимание к важным моментам или визуально разделить content.

 

Примеры синтаксиса и результата
Способ 1: Непосредственная вставка (наиболее прямой)

Просто скопируйте эмодзи из любого источника и вставьте в текст.

Исходный код (Markdown):
```
# Добро пожаловать! 🎉

Отличная работа! 👏 
Не забывайте делать бэкапы! 💾
``` 

Как это будет выглядеть после обработки:

Добро пожаловать! 🎉
Отличная работа! 👏
Не забывайте делать бэкапы! 💾

 

Способ 2: Кодовые последовательности (более универсальный)

Использование синтаксиса ```:имя_эмодзи:```.

Исходный код (Markdown):

Настроение сегодня ```:sun_with_face:``` ```:sparkles:```

Внимание! ```:warning:``` Этот раздел важен!

```:rocket:``` Запускаем проект!
```:white_check_mark:``` Задача выполнена
```:x:``` Ошибка

Настроение сегодня :sun_with_face: :sparkles:

Внимание! :warning: Этот раздел важен!

:rocket: Запускаем проект!

:x: Ошибка

# Поддержка HTML

Markdown изначально задумывался как удобный синтаксис для преобразования в HTML. Поэтому почти все HTML-теги можно использовать прямо внутри Markdown-документа. Это позволяет расширять возможности разметки там, где средства самого Markdown недостаточны.

 

Как это работает?
Вы можете вставлять HTML-теги непосредственно в текст Markdown. Парсер обработает Markdown-разметку, а HTML-теги останутся как есть и будут интерпретированы браузером (или другим рендерером) уже как валидный HTML.

 

Почему это важно?
HTML предоставляет полный контроль над оформлением и поведением контента. Это ваш "аварийный выход", когда нужно:

Добавить сложную таблицу с объединенными ячейками (colspan, rowspan).

Использовать специфические стили (```<span style="color: red;">```).

Вставить видео, аудио или iframe.

Реализовать детали, которые не покрываются синтаксисом Markdown.

 

Примеры синтаксиса и результата
Исходный код (Markdown):
```
Это **жирный текст** в Markdown.

А это <span style="color: red; font-weight: bold;">красный жирный текст</span> в HTML.

Markdown-ссылка: [GitHub](https://github.com).

HTML-ссылка: <a href="https://github.com" target="_blank">GitHub (откроется в новой вкладке)</a>.
```

Это **жирный текст** в Markdown.

А это <span style="color: red; font-weight: bold;">красный жирный текст</span> в HTML.

Markdown-ссылка: [GitHub](https://github.com).

HTML-ссылка: <a href="https://github.com" target="_blank">GitHub (откроется в новой вкладке)</a>.
