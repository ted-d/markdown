# Выделение текста
Это *курсив* с помощью звездочек.
Это _тоже курсив_ с помощью подчеркиваний.

Это **жирный** текст с помощью звездочек.
Это __тоже жирный__ текст с помощью подчеркиваний.

А это ***жирный курсив*** с помощью звездочек.
И это ___тоже жирный курсив___ с помощью подчеркиваний.

Формально можно и **_комбинировать_** оба символа (хотя, это ухудшает читабельность текста).

⚠️ Важно: символы выделения должны *примыкать к тексту* без пробелов.

# Разделительные линии

Конспект урока
Горизонтальная линия (или разделитель) — это элемент, который визуально разделяет разные разделы документа длинной горизонтальной чертой. Она помогает структурировать контент, обозначая крупные смысловые блоки, и улучшает визуальное восприятие текста.

 

Как это работает?
Для создания линии в Markdown используются три одинаковых символа подряд на отдельной строке. Пробелы между символами не обязательны, но улучшают читаемость исходного кода. Допустимые символы: дефисы (-), звёздочки (*) или знаки подчёркивания (_).

 

Почему это важно?
Разделительная линия — это мощный визуальный инструмент для организации контента. Она особенно полезна в длинных документах, где помогает читателю понять, где заканчивается одна тема и начинается другая, не перегружая текст лишними заголовками.

 

Исходный код (Markdown):
Это первый раздел документа.

***

Вот так создается простая разделительная линия.

---

Можно использовать дефисы (самый популярный вариант).

___

Или нижние подчеркивания.

- - -

Между символами можно ставить пробелы.

* * *

Это тоже будет работать.

Второй раздел документа после линии.
 

Важное примечание: Для гарантированного и предсказуемого создания линии настоятельно рекомендуется делать пустые строки до и после строки с символами. Хотя многие парсеры Markdown (особенно современные) справятся и без них, отсутствие пустых строк может привести к непреднамеренному результату.

# Программный код

Markdown предоставляет мощные инструменты для вставки программного кода, обеспечивая его корректное и читаемое отображение. Поддержка кода разделяется на три основные области:

Строчные (inline) фрагменты — для коротких упоминаний в теле текста.

Многострочные блоки — для демонстрации больших примеров.

Подсветка синтаксиса — автоматическое раскрашивание кода в зависимости от языка программирования для улучшения восприятия.

 

## Как это работает?

Строчные фрагменты обрамляются одинарными бэктиками (`).

Блоки кода обрамляются тройными бэктиками (```) или создаются через отступ (4 пробела или 1 табуляция).

Подсветка синтаксиса активируется указанием имени языка программирования сразу после открывающих тройных бэктиков (напр., ````python`).

 

## Почему это важно?
Корректное оформление кода — признак качественной технической документации. Оно предотвращает непреднамеренную интерпретацию символов Markdown, сохраняет исходное форматирование и многократно повышает читаемость и удобство восприятия сложных примеров.

2. Многострочные блоки кода (с подсветкой синтаксиса)

Назначение: Большие примеры кода, сохраняющие форматирование и переносы строк.
Синтаксис: Тройные бэктики + указание языка.

Исходный код (Markdown):

Пример кода на Python с подсветкой:
```python
def factorial(n):
    """Вычисляет факториал числа n."""
    if n == 0:
        return 1
    else:
        return n * factorial(n - 1)

result = factorial(5)
print(f"Факториал 5 равен {result}")
```
